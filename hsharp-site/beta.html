v0.5 改了什么

从 v0.4 升上来需要知道的 5 件事。

01

多线程 DZZW 调度器

协程现在可以跑在工作池上，CPU 密集任务的吞吐提升 2.4×（raytrace benchmark）。通过 @parallel 装饰器显式启用。

perf +140%
02

async / await 原生语法

不再需要 callback(then) 链。新的 async 关键字 + await 表达式，编译期静态分析，比 callback 链错误率低 60%。

language
03

标准库 -40% 体积

把 std/legacy 整个挪到 contrib/，H# 启动器内存占用从 32 MB 降到 19 MB，冷启动时间从 1.4s 降到 0.7s。

size -40%
04

ZZW Code 1.3

IDE 侧栏新增 AI Debug 视图：断点触发时直接把栈帧发给 ZZW Code Agent，让它主动提出「我看到这里可能是因为 X」的解释。带 3 周 free trial。

ide
05

诊断信息更详细

编译器错误从 1 行扩展为多行 ^^^^ 高亮 + 修复建议。配合 h# --explain E0308 可以拿到通俗解释，零联网、零数据上报。

devx

与 v0.4 共存安装

beta 版的二进制被命名为 h#050，不会覆盖你已有的 h#。需要把 h#050 单独加到 $PATH。

chmod +x hsharp-0.5.0-beta.1-macos-arm64.hps
sudo mv hsharp-0.5.0-beta.1-macos-arm64.hps /usr/local/bin/h#050
h#050 --version    # → H# v0.5.0-beta.1

回滚到 v0.4

只需要 sudo rm /usr/local/bin/h#050，v0.4 不会被影响。

安装指引+

beta 启动器会被命名为 ZZW Code Beta.app，与 v0.4 启动器在「应用程序」文件夹下并存。运行时不会串 v0.4 项目的状态。

启动时若提示「未识别开发者」，到「系统设置 → 隐私与安全性」点击「仍要打开」。

h# build compiler/main.hto -o h#050
