01

安装与环境

H# 的工具链以「单二进制 + 解释器 + 编译器」三件套形式分发。在 macOS / Linux / Windows 上都能直接运行 —— 不需要任何运行时依赖。

系统要求：macOS 12+ / Ubuntu 20.04+ / Windows 10+ · 仅需 50 MB 磁盘。
bash — install .sh 
# macOS · Homebrew
brew install hsharp/tap/hsharp

# Linux · one-line installer
curl -fsSL https://hsharp.dev/install.sh | sh

# Windows · winget
winget install hsharp

安装完成后，h# --version 会打印 h# 0.4.2 (self-hosting)。其中 self-hosting 表示这个二进制本身就是用 H# 编译出来的 —— 它编译自身，没有任何外部编译器参与。

02

第一个 H# 程序

新建一个 hello.hto 文件，写下你第一段 H# 代码：

hello.hto .hto 
// H# 的入口是 main 函数
fn main() {
  let name = "H#"
  print("Hello, " + name + "!")
}

两种运行方式：

  • 解释执行：h# hello.hto —— 最快反馈，适合开发。
  • 编译执行：h# build hello.hto → ./hello —— 生成原生字节码，启动更快。
小贴士：所有 .hto 文件用 h# run 直接运行；想看 AST 加 --ast，想看字节码加 --bc。
03

变量与类型

H# 是一门静态推断语言 —— 多数时候你不需要显式标注类型。变量用 let 声明，常量用 const。

types.hto .hto 
fn main() {
  let n: int = 42
  let pi: float = 3.14
  let msg: str = "hello"
  let ok: bool = true
  let arr: [int] = [1, 2, 3]
  let pair: (str, int) = ("age", 18)

  const MAX = 1000
}
类型示例说明
int4264 位有符号整数
float3.1464 位 IEEE-754
str"hi"不可变 UTF-8 字符串
booltrue布尔
T?NoneOption 类型
[T][1, 2]动态数组
map[K]V{}哈希表
04

控制流

H# 的控制流和 C 系语言非常接近 —— 但用 match 替代 switch，用 ? 替代 try/catch 的繁琐。

flow.hto .hto 
fn classify(n: int) -> str {
  match n {
    0             => return "zero"
    in 1..9      => return "small"
    in 10..99    => return "medium"
    _              => return "huge"
  }
}

fn main() {
  for i in 0..3 {
    if i == 1 { continue }
    print(classify(i))
  }
}
05

函数与闭包

H# 的函数是一等公民 —— 可以作为参数传递，可以赋值给变量，可以从函数中返回。闭包自动捕获外层变量。

fn.hto .hto 
fn make_adder(step: int) {
  return fn(n) { n + step }
}

fn main() {
  let add10 = make_adder(10)
  print(add10(5))   // 15

  let nums = [1, 2, 3, 4]
  let doubled = nums.map(fn(x) { x * 2 })
  print(doubled)          // [2, 4, 6, 8]
}

结构体与枚举

H# 没有「类」，但 struct + impl + enum 三件套足以表达绝大多数领域模型。

model.hto .hto 
struct User {
  name: str
  age:  int
}

enum Event {
  Click(x: int, y: int)
  Key(code: str)
  Quit
}

impl User {
  fn greet(self) {
    print("hi, " + self.name)
  }
}
07

错误处理

H# 用 Result<T, E> 替代异常 —— 错误是值，不是控制流。? 操作符让链式调用保持可读。

errors.hto .hto 
fn read_config(path: str) -> Result<str, str> {
  let raw = fs.read(path)?
  return Ok(raw.trim())
}

fn main() {
  match read_config("h.toml") {
    Ok(c)  => print(c)
    Err(e) => print("oops: " + e)
  }
}
08

并发：DZZW 原语

H# 的并发模型叫 DZZW —— 灵感来自 Go 的 channel 和 Erlang 的 actor，但用 spawn + chan 两个关键字就够。

dzzw.hto .hto 
fn producer(out: chan<int>) {
  for i in 0..5 { out.send(i) }
  out.close()
}

fn main() {
  let ch = chan<int>(2)
  spawn(producer, ch)
  for v in ch { print(v) }
}
注意：DZZW 调度器使用 M:N 协程模型，单进程可承载数十万并发任务；但发送阻塞在容量有限的 chan 上时不会自动扩容。
09

字节码与虚拟机

每一个 .hto 文件都会被编译成一组 42 条字节码指令 之一，序列化成 .hbc 文件后由 stack-VM 解释执行。

compiled.hbc — disassembly .hbc 
0000  PUSH_I64     42
0002  STORE        n@0
0004  PUSH_STR     "hello"
0006  STORE        msg@1
0008  LOAD         n@0
000a  PRINT
000b  HALT

想看完整指令集，运行 h# opcodes。每条指令都用一个 8-bit 操作码 + 最多两个 64-bit 操作数编码 —— 整个字节码可序列化，可跨进程加载。

自举：H# 写 H#

v0.4 起，H# 解释器本体（~8000 行）已经用 H# 自身重写并自举。这意味着：

  • stage0：用 H# 写的 H# 编译器，编译出 stage1 二进制；
  • stage1：用 stage1 重新编译自身，产出 stage2；
  • stage2：再次自举，验证 stage1 == stage2 比特完全一致 —— 编译器「站住了」。
build — bootstrap .sh 
# Reproducible bootstrap from source
h# build --stage=0 compiler/main.hto  -o h0
h# build --stage=1 h0 compiler/main.hto -o h1
h# build --stage=2 h1 compiler/main.hto -o h2
diff h1 h2   # must be empty

ZZW Code 实战

ZZW Code 是 H# 的官方 IDE —— 把上面所有内容变成「在编辑器里直接和 AI 协作」的工作流。

  1. 安装：从 下载页 拉一份 ZZW Code，安装时附带 H# 工具链。
  2. 配置 API Key：⌘ , → Providers → 填入 Endpoint / Key / Model。OpenAI 兼容即可。
  3. 打开项目：⌘ O → 选 .hto 文件。ZZW Code 会自动注入 H# 知识到系统提示词。
  4. 让 AI 干活：在 AI 聊天面板输入 "解释 parser.hto" → ZZW AI 读取当前文件 → 返回可一键应用的补丁。
  5. SOLO 模式：⌘ ⇧ S 把目标交给 Agent —— 拆任务、跑测试、改 bug 一气呵成。
